# Recruitment Management Module (موديول إدارة التوظيف)

## 📋 Overview

provides a complete recruitment lifecycle management system within the HR API. It covers job postings, candidate applications, and automated interview scheduling with strict business logic and hierarchical access control.

## 🏗️ Architecture

The module is built using the standard project layering:

- **Models:** `Job`, `Candidate`, `Interview`.
- **Enums:** `JobStatusEnum`, `CandidateStatusEnum`, `InterviewStatusEnum`.
- **Services:** `JobService`, `CandidateService`, `InterviewService`.
- **Repositories:** Following the Repository Pattern with Interfaces.
- **DTOs:** Structured data for creation and updates.

## 🛠️ Key Components

### 1. Job Management (`/api/recruitment/jobs`)

- **Job Status:** Jobs can be `DRAFT`, `PUBLISHED`, or `CLOSED`.
- **Application Logic:** Candidates can only apply to jobs with `PUBLISHED` status.
- **Validation:** Includes checks for job existence and active status before accepting applications.

### 2. Candidate Tracking (`/api/recruitment/candidates`)

- **Status Flow:** `PENDING` → `INVITED_TO_INTERVIEW` or `REJECTED`.
- **Status Persistence:**
    - Uses Laravel's native **Enum Casting** for robust data integrity.
    - Status can only be updated if the current state is `PENDING`. Decisions are locked once made.
- **Dynamic Field Mapping:** Accepts both `application_status` and `status` to ensure compatibility across different UI components/modules.

### 3. Interview Scheduling (`/api/recruitment/interviews`)

- **Automated Trigger:** When a candidate is updated to `INVITED_TO_INTERVIEW`, an interview record is automatically created in the `ci_rec_interviews` table.
- **Status Restrictions:** Interview status can only be updated while it is in the `NOT_STARTED` state.
- **Data Integrity:** Maintains links between jobs, candidates, and interviewers.

## 🌍 Globalization & Normalization

To support a seamless Arabic/English experience, the module implements **Status Normalization**:

- **Arabic Handling:** Normalizes variations in Arabic letters (ة/ه، أ/إ/آ، ى/ي) during input.
- **Flexible Input:** Accepts English keywords (e.g., `rejected`) or Arabic labels (e.g., `مرفوض`) and maps them to the correct internal Enum values.

## 🔐 Business Rules & Logic

### 🛡️ Access Control

- **Hierarchy Check:** Users (Staff) can only see candidates and interviews for employees/jobs within their hierarchy and department constraints (via `SimplePermissionService`).
- **Isolation:** Strictly isolated by `company_id`.

### ⚡ Validation Rules

- **Conditional Fields:** While updating a candidate to `INVITED_TO_INTERVIEW`, fields like `interview_date`, `interview_time`, and `interviewer_id` are **REQUIRED**.
- **Optional Reject:** In case of `REJECTED`, these fields are **OPTIONAL**, allowing for a quick one-click rejection process.

## 📡 API Endpoints Summary

| Method | Endpoint                                  | Description                                   |
| ------ | ----------------------------------------- | --------------------------------------------- |
| GET    | `/api/recruitment/jobs`                   | List job postings with pagination.            |
| POST   | `/api/recruitment/jobs/apply`             | Apply for a job (Resume upload required).     |
| GET    | `/api/recruitment/candidates`             | List candidates with status/job filters.      |
| POST   | `/api/recruitment/candidates/{id}/status` | Update candidate status (Triggers Interview). |
| GET    | `/api/recruitment/interviews`             | List scheduled interviews.                    |
| POST   | `/api/recruitment/interviews/{id}/status` | Finalize interview decision (Success/Reject). |
